  • 1 maven是什么？

  官方介绍：Apache Maven is a software project management and comprehension tool.Based on the concept of a project object model (POM), Maven can manage a project’s build, reporting and documentation from a central piece of information.

  maven是一个软件项目管理和理解的工具，这里的management管理就是指maven帮助我们对于jar包的添加删除管理，配置文件的管理，各个模块之间关系的管理等，comprehension理解说的就是你通过maven构建出来的项目，具有一套标准的项目框架，这样就能能更好的帮助你理解这个项目。

  maven是基于POM：项目对象模型的，maven由java语言编写的，所以maven把它管理的项目当成一个对象来进行管理，管理对象当然就是管理对象里边的属性和行为，这里被管理的项目就通过一个pom.xml文件来告知maven需要管理这个项目的哪些东西。
  然后,maven通过一个依赖管理的模块，对这个项目对象进行资源的注入（主要是jar包），以及一个生命周期的模块（由maven自带的多种插件构成）对项目对象进行编译、运行、测试、打包等操作。
  资源的拉取，前面说了maven可以对项目进行资源的注入，具体是将本地（自己电脑内，安装maven时会指定仓库路径）的maven仓库里边的jar包注入到项目，而在公司开发时，许多人使用的仓库是公司的仓库，也就是私服（存储在网络的指定路径下，需要公司指定的内网访问下载）里边可能会有许多公司内部自己的jar包资源。大部分资源都来源于maven的中央仓库。但是中央仓库访问也很慢，一般是通过国内的镜像资源下载的。

  • Maven前期

  • 1 maven的坐标

  什么是maven的坐标？前面讲了，在项目中需要各种依赖的jar包资源，而这些jar包导入到maven构建的项目中就是通过坐标的方式导入的，在pom.xml中写坐标给maven用，maven就可以通过坐标自动下载这些资源，一般是先在本地仓库查找下载，再层层递进到中央仓库。

  maven坐标的格式

groupId: 该项目隶属的组织名称/通常域名反写
artifactId：项目名称/通常模块名称
version：版本号
packaging： 打包方式

  • 2 本地仓库配置

自己建一个文件夹放在指定位置作为本地仓库，通过修改conf中的settings.xml中的localRepository标签为本地仓库地址即可。默认的仓库是在c盘的用户文件下的.m2文件中。

  • 3 远程仓库配置

默认的中央仓库的配置在lib文件夹下的pom-4.00xml文件中，其配置的是国外的服务器仓库网址，下载速度非常慢。所以我们需要进行修改，在settings.xml文件中的mirror标签内配置镜像仓库。配置如下：

<mirror>
    <!--该镜像的标识符，用于区分不同的镜像 -->
 	<id>nexus-aliyun</id>
    <!--对哪一个仓库进行镜像 -->
    <mirrorOf>central</mirrorOf>
    <!--镜像名称 随意-->
    <name>Nexus aliyun</name>
    <!--镜像URL-->
    <url>http://maven.aliyun.com/nexus/content/groups/public</url>
</mirror>

  • 1 配置maven环境

  idea中创建一个新的空项目，对该项目环境进行简单配置如下：

  • 第一项

    project Settings里边选定项目所使用的SDK(Software Development Kit：软件开发工具包)，常用的是1.8,11,17的版本。

  • 第二项

    file -> settings 搜索maven（路径为：Build、Execution、Deployement选项里边的BuildTools里边的maven），最下边可以看到三条路径修改符，Maven home path：所使用的maven路径，修改成自己的maven路径；User settings file:修改成自己maven的settings.xml文件路径；Local repository：本地仓库路径-修改成自己创建的本地仓库路径。 最后检查自己的maven runner里边的jre版本是否对应，以及Compiler中的java compiler的版本是否对应。

    注：以上修改只对于当前项目生效，全局生效（对以后创建的任意maven项目都生效）的话，需要close project后，在Customize -> All settings里边如上修改即可

  • 2 创建maven项目的两种方式

  • 第一种 直接创建

    File ——> settings ——>Project structure ——>module ->点击+号——>new module

  • Maven基础使用

  • 1 maven依赖管理

  依赖配置：配置依赖就是通过在pom文件种添加对应依赖的坐标信息，刷新后即可将所需要的jar包添加到项目中。配置依赖也可以将项目中不同的模块进行配置，只需要配置对应的坐标即可。标签：denpendencies里边多个dependency

  依赖传递：给一个模块进行依赖配置（使用jar包或者使用其他模块所开发的工具）时候，被使用的依赖可能会有其他依赖，这些依赖也一并给了这个项目。在项目中直接配置的为直接依赖，另外就是间接依赖。

当项目中包含多个依赖时，可能发生依赖传递的冲突（就是多个相同依赖但版本不同）：
优先情况：
1 以项目为根，直接依赖往后一级一级优先级降低，前面的覆盖后边的。
2 相同等级，先配置的覆盖后配置的

  可选依赖：对外隐藏当前所依赖的资源。就是说，你给别人用的时候，可以把你自己用的东西给隐藏。
将坐标的optional标签中添加ture即可隐藏

  排除依赖：你用别人的依赖的时候，如果你不想要别人依赖中依赖的其他依赖，就可以进行排除。 在依赖的坐标添加如下代码

<exclusions>
    <exclusion>
        <groupId>***</groupId>
        <artifcatId>***</artifactId>
    </exclusion>
</exclusions>

  依赖范围：对于每一个添加到项目的jar包依赖，都有一个使用范围。通过scope标签进行设定 （默认为compile）

简单的Linux命令

1 基本格式

  linux命令的基本格式为：

command [-options] [parameter]
命令本体 命令行为细节控制 命令指向目标

第一个参数为命令本身必须有，第二个参数为命令选项可以进行组合，第三个参数为命令指向的目标

2 各种重要命令

  • ls命令

ls  [-a、-l、-p]  [指定文件夹]
即参数list，展开文件夹的目录将其显示，-a可以展开隐藏的文件夹，-l以列表展示并且
会展示更多细节，如文件的权限和日期等，-h还可以展示文件的大小
